01 System Overview
Architecture
8bit
CPU Family
Z80
Boot Medium
ROM
Input Device
Keyboard

OM-8 は Z80 プロセッサを核とする 8 ビット・パーソナルコンピューティングシステムである。 ビルドインのビットマップフォントを用いたテキスト出力機構、キーボード入力、および ROM ブートに対応し、 汎用的かつ本格的なコンピューティング用途への展開を目的として設計された。

本仕様書はハードウェア構成、メモリマップ、入出力インタフェース、および起動シーケンスについて定義する。

02 Central Processing Unit
PROCESSOR Zilog Z80 / 互換品
ARCHITECTURE 8-bit / 16-bit アドレス空間
CLOCK FREQUENCY 最大 10 MHz（設計値）
実装クロックソースに依存。クリスタル発振器を推奨。
REGISTERS A, B, C, D, E, H, L, F, IX, IY, SP, PC, 予備セット
ADDRESS BUS 16-bit（アドレス空間 64 KB）
DATA BUS 8-bit
INSTRUCTION SET Z80 ネイティブ命令セット（拡張プレフィックス CB / DD / ED / FD 含む）
INTERRUPT INT / NMI（モード 0 / 1 / 2 対応）
I/O SPACE 256 ポート（IN / OUT 命令）
03 Memory Map
0x0000 – 0x3FFF ROM ブート ROM（16 KB）
電源投入後、Z80 は 0x0000 番地から実行開始。ROM に格納されたファームウェアを読み込む。
0x4000 – 0x7FFF VRAM テキストビデオRAM（16 KB）
文字コードおよびビットマップフォントデータを格納。ディスプレイコントローラが参照する。
0x8000 – 0xFFFF RAM 汎用ワークRAM（32 KB）
スタック、変数、プログラム展開領域として使用。

04 Display & Graphics
OUTPUT TYPE テキスト出力（ビットマップフォント描画）
DISPLAY RESOLUTION 640 × 400 ピクセル（推奨）
実装ディスプレイコントローラに準じる。
TEXT MODE 80 列 × 25 行
FONT ビルドイン 8×16 ビットマップフォント（ROM 内蔵）
ASCII (0x20–0x7E) + 拡張文字対応。フォントデータは ROM の固定アドレスに配置。
COLOR モノクロ（前景 / 背景 2 色）
オプションで 4 色パレット拡張可能（実装依存）。
CURSOR ソフトウェアカーソル（ブロック / アンダーライン 切替）
REFRESH RATE 60 Hz（VSYNC 割込みにより Z80 に通知）
SCROLL ハードウェアスクロールまたはソフトウェア VRAM コピー
VRAM ACCESS CPU による直接書き込み（0x4000 番地以降）
NOTE — GRAPHICS SCOPE 本システムのグラフィックス機能はテキスト出力に特化しており、スプライトや任意ピクセル描画機能は提供しない。 高度なグラフィックス用途が必要な場合は拡張ボードの使用を検討すること。
05 Keyboard Input
INTERFACE PS/2 または 独自マトリクスキーボード（実装依存）
CONTROLLER 専用キーボードコントローラ IC（I/O ポート経由で Z80 と通信）
I/O PORT ポート 0x00（ステータス）/ 0x01（データ）
SCAN CODE 1 バイトスキャンコード方式
キー押下・リリースを区別。修飾キー（Shift / Ctrl / Alt）対応。
BUFFER 16 バイト FIFO ソフトウェアキーバッファ（ROM ファームウェア管理）
INTERRUPT INT ライン接続。キー入力時に Z80 割込みを発生させる方式を標準とする。
POLLING 割込み非使用時はポーリングによるアクセスも可能。
06 Boot Sequence
  • POWER ON — リセット信号がアサートされ、Z80 の PC が 0x0000 に初期化される。
  • ROM FETCH — 0x0000 番地の ROM ファームウェアが実行開始。スタックポインタ・レジスタを初期化。
  • HARDWARE INIT — VRAM クリア、ディスプレイコントローラ初期化、キーボードコントローラリセットを順次実行。
  • SELF TEST — RAM 書き込み・読み出し検査（0x8000–0xFFFF 領域）。エラー検出時は VRAM にエラーコードを表示して停止。
  • FONT LOAD — ROM 内蔵ビットマップフォントをフォントテーブルとして参照可能な状態にセット。
  • FIRMWARE LAUNCH — メインインタプリタ / モニタプログラムへジャンプ。コマンドプロンプトを表示し、ユーザー入力待ち状態に遷移。

07 Audio

OM-8 は音声出力機能を搭載しない。オーディオ回路、サウンドチップ、およびスピーカー出力は 本リビジョンの設計スコープに含まれていない。

将来の拡張ボードによって音声出力機能を追加することは妨げないが、 本体回路基板にオーディオ関連の部品・端子は実装されない。

DESIGN RATIONALE 音声機能の省略は、回路の簡素化・コスト低減・電磁ノイズ低減を目的とした設計判断である。 テキストベースのコンピューティングに特化した用途においては影響がないと判断した。
08 Electrical & Physical
SUPPLY VOLTAGE +5 V DC（±5%）
TYPICAL CURRENT 300–500 mA（実装依存）
LOGIC LEVEL TTL / CMOS 互換（3.3 V 対応品は変換レベルシフタを要する）
BUS EXPANSION Z80 バスを外部コネクタに引き出し（アドレス / データ / 制御信号）
FORM FACTOR カスタム PCB（寸法は実装設計に依存）
OPERATING TEMP 0 °C – 50 °C
